# Day 4: Introduction to SQL & Environment Setup

Welcome to Day 4! This section transitions from the theoretical concepts of Database Management Systems (DBMS) into practical, hands-on application using SQL (Structured Query Language) and Oracle tools.

## Topics Covered

### 1. Introduction to SQL
* **Concept:** SQL is the standard programming language used to communicate with relational databases.
* **What you will learn:** The foundational syntax and purpose of SQL, including how it is used to query, insert, update, and delete data, as well as how it enforces the relational models and constraints discussed in previous days.

### 2. Environment Setup (Oracle & SQL Developer)
* **Concept:** To write and execute SQL, you need a database engine and an interface to interact with it.
* **Installing Oracle:** This is the installation of the core Relational Database Management System (RDBMS) server. It acts as the "engine" that stores and processes the data locally on your machine.
* **Installing SQL Developer:** This is the Graphical User Interface (GUI) client. Instead of typing commands into a raw terminal, SQL Developer provides a visual environment to write queries, view table structures, and manage the database.
* **Connection:** You will learn how to configure SQL Developer to successfully communicate with your newly installed Oracle database engine.

### 3. Database Administration Basics
* **Creating a New User:** Databases use strict access controls. You will learn how to create a dedicated user account within the database, granting it the specific privileges needed to create tables and manipulate data safely without altering system-level configurations.
* **Creating a Sample Database:** Before writing complex queries, you need a sandbox environment. This involves executing scripts or commands to build a fresh, dummy database (often containing sample tables like `Employees`, `Departments`, etc.) so you can safely practice your SQL skills.
